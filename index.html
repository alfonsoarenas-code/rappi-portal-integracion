Webhook — orden nueva POST

POST /api/v1/orders/webhook
X-Signature: HMAC-SHA256

{
  "event": "order.new",
  "order_id": "ORD-4782",
  "items": [...],
  "total": 60000,
  "prep_time_estimate": 18
}

Responder 200 OK en menos de 5 seg. Sin ACK = fallback.

Confirmar orden PATCH

PATCH /api/v1/orders/{id}/status

{
  "status": "accepted|rejected
             |preparing|ready",
  "channel": "pos|whatsapp
              |sms|ivr"
}

Idempotente con Idempotency-Key.

Cascada de fallback

1. POS webhook (30s) → 2. WhatsApp API (90s) → 3. SMS + IVR (60s) → 4. Auto-pausa. Respuesta en cualquier canal cancela timers.

Sync de menú bidireccional

POS = fuente de verdad. Cambios de precio y disponibilidad vía menu.sync webhook. Items agotados desde POS se reflejan en la app en <60 seg.

Auto-pausa de tienda

Si todos los canales fallan: store.status → paused. Clientes no ven la tienda. Reactivación: aliado confirma desde cualquier canal.

POS compatibles Colombia

Nativo: Siigo Gastrobar, Loggro Restobar, Vendty. Vía middleware: Alegra, Conexión POS, Square, Lightspeed. Middleware: Deliverect, Otter, Chowly.
